## リアルタイムログ監視機能（Issue #45）

### 概要
`orch run` で実行中の Claude/OpenCode の出力を、別ターミナルからリアルタイムで監視できるようにする。

### 期待する動作
```bash
# ターミナル1: 実行
orch run --issue 42 --auto

# ターミナル2: リアルタイム監視
orch logs --task <task-id> --follow
# → Claude/OpenCode が出力しているテキストがリアルタイムで流れる
```

### 実装方針案

#### 方針1: ファイルベース（シンプル）
- `LogWriter` が書き出すログファイルを `tail -f` 的に読み取り
- `fs.watch()` または polling で新しい行を検出
- 既存の `LogWriter` / `LogStreamer` を拡張

#### 方針2: Unix ソケット / Named Pipe
- 実行プロセスが Unix ソケットに出力
- `orch logs` がソケットに接続してストリーミング
- より低レイテンシーだが実装が複雑

### CLI インターフェース
```bash
# 実行中のタスク一覧から選択
orch logs --follow

# 特定タスクを監視
orch logs --task <task-id> --follow

# 最新のタスクを監視
orch logs --latest --follow
```
